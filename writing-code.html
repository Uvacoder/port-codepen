On writing code

I've been doing it since 2007. I mean, that's when I wrote down my first ever HTML, which was just a <br />. But I got remarkably better at it over time. Until 2013, it was just something I did for fun. In 2013, I wanted to work, and I wanted to become a web designer but I couldn't find a work for myself in that path, so I ended up working as a software developer. However, I still enjoy thinking about user interface and experience design.

In my first job, I mostly wrote CoffeeScript code. Then, in my later jobs, I moved back to writing plain JavaScript. In the recent years, I sometimes needed to write TypeScript. I like them all *Scripts. But I love the CSS. There's not a technical reason why I love CSS over all other types of code. I think it's rather because I spent so much time writing CSS for fun when I was a teenager.

Shortly after I started my first job, I discovered that merely writing code won't help making enough money. So, I started reading and learning about topics like git, software design patterns, clean code authorship practices, algorithms and data structures, databases, accessibility, performance, security, color theory, interaction design, various frameworks etc. It was and is a nice journey exploring all these topics and more.

Lastly, I'm an enthusiast of a text editor, that's used by many people to write code, called Vim. It has this nice good-old-days vibe going on compared to other popular text editors, I like it. In fact, I'm writing these lines in Vim.
